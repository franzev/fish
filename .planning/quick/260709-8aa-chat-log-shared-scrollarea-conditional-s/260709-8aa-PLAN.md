   `useStickToBottom({ messages, currentUserId }: { messages: LocalMessage[]; currentUserId: string })` returning `{ viewportRef, showNewMessages, scrollToBottom }` where:
   - `viewportRef: RefObject<HTMLDivElement | null>` — caller passes to ScrollArea's viewportRef.
   - `scrollToBottom(behavior?: ScrollBehavior)` — scrolls viewport to `scrollHeight`; default behavior "smooth" unless `window.matchMedia("(prefers-reduced-motion: reduce)").matches`, in which case force "auto". Also clears the pill state.
   - Internal `isNearBottomRef`: updated from a passive "scroll" listener on the viewport: near bottom when `scrollHeight - scrollTop - clientHeight <= 100` (threshold ~100px). When the listener detects the user is back at bottom, clear `showNewMessages`.
   - Initial mount effect (layout effect preferred): jump to bottom instantly (`behavior: "auto"` / direct `scrollTop = scrollHeight`) — no animation on first paint.
   - Message-growth effect: track previous message count + last message in refs. When `messages.length` grows:
     - if last message `senderId === currentUserId` → `scrollToBottom()` (own send always sticks; also covered by explicit call in Task 2, harmless double);
     - else if near bottom → `scrollToBottom()`;
     - else → `setShowNewMessages(true)` and DO NOT scroll.
   - A `ResizeObserver` on `viewport.firstElementChild` (the content): when content grows while `isNearBottomRef.current` is true, re-pin to bottom with `scrollTop = scrollHeight` (instant — keeps stickiness for typing indicator / late-loading content without animating). Disconnect on cleanup.
   - Guard all window/DOM access for SSR (effects only; hook file starts without "use client" since it is only imported by a client component — adding "use client" is fine too).
