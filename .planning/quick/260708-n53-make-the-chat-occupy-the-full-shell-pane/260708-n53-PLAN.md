# Quick Task 260708-n53: Full-bleed, viewport-locked chat

## Context

Follow-up to 260708-mjs (unboxed chat). The box is gone but the shell still
constrains the chat: `<main>` in app-shell.tsx caps width at `max-w-content`
with page padding, and the shell root uses `min-h-dvh`, so nothing bounds
height — the whole page scrolls instead of just the message log. The user
wants the chat edge-to-edge in the pane (Discord idiom), simpler tailwind
class strings, and redundant wrapper divs removed.

## Tasks

### Task 1 — Immersive main for the chat route

**Files:** `apps/web/components/shell/app-shell.tsx`, `apps/web/components/shell/app-shell.test.tsx`

**Action:** AppShell already reads `usePathname()` for nav state; derive
`immersive = isActivePath(pathname, "/chat")`. Root div: `h-dvh` when
immersive (else `min-h-dvh`) so the pane bounds height. Main: immersive gets
`flex min-h-0 w-full flex-1 flex-col pb-mobile-nav-offset md:pb-0` (no
max-w-content, no page padding — composer still clears the fixed mobile
nav); other routes keep the centered D-10 column unchanged. Add a test for
the immersive variant; the existing D-10 test (pathname /home) still holds.

### Task 2 — Simplify chat-client section and drop redundant wrappers

**Files:** `apps/web/app/(authenticated)/chat/chat-client.tsx`,
`apps/web/components/chat/chat-container/chat-container.tsx`

**Action:** Section becomes `flex min-h-0 w-full flex-1 flex-col` — drops
the `min-h-chat-container-demo` demo token and redundant `bg-bg` (it sits on
the page background now); with the bounded pane the existing
`flex-1 overflow-y-auto` log finally scrolls internally. Replace the
`<div className="border-t border-border px-md pt-sm"><Alert/></div>` notice
wrapper with `Alert className` (Alert accepts HTMLAttributes). Kit
ChatContainer drops its redundant `bg-bg`.

**Verify:** `pnpm build` + vitest pass; preview shows the chat filling the
pane with only the log scrolling.
