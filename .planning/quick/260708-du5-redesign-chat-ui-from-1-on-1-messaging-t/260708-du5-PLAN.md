# Quick Task 260708-du5: Community-room chat UI redesign

## Context

The product pivoted from personal 1-on-1 messaging to a community chat platform.
The data layer is already in the working tree (uncommitted): `ClientChatData.kind`
("direct" | "community"), `title`/`subtitle`, `senderDisplayName`, the demo
community migration (`0014_demo_community_conversation.sql`), and core.ts
preferring the fixed community room. What remains is the UI/IA redesign.

Design constraints (from `sketch-findings-fish` + AGENTS.md): pure monochrome,
calm and choice-free, one primary action per screen, 56px targets, copy never
scolds, reward-only. The community idiom is Discord-like *structure* (flat
left-aligned author rows, room header, member count) rendered in FISH's calm
monochrome voice — not Discord's density or server/channel sprawl.

## Tasks

### Task 1 — Navigation shell: Messages → Community

Files: `apps/web/components/shell/app-shell.tsx`, `apps/web/components/shell/app-shell.test.tsx`

- Rename the `/chat` nav item label from "Messages" to "Community" for both
  client and coach navs; switch its icon to `IconUsersGroup` (community of
  people) so the coach's `IconUsers` "Clients" item stays distinct.
- Update shell tests that assert the "Messages" label.

Done when: nav reads Home · Community · Profile (client) and Clients · Community (coach); tests pass.

### Task 2 — Community room feed + header in ChatClient

Files: `apps/web/app/(authenticated)/chat/chat-client.tsx`

- Header (community kind): replace the DM presence line with a member count
  ("N members") derived from readStates ∪ message senders ∪ current user;
  keep room title/subtitle, search, unread badge.
- Feed (community kind): Discord-idiom flat rows — every message (own included)
  left-aligned with an avatar column; `MessageMeta` (author + timestamp) at
  each group start; flat monochrome text instead of mine-right primary bubbles.
- Delivery semantics: in a room, drop per-participant "seen" ticks; keep
  sending/failed + Retry (calm copy).
- Community empty-feed copy: "No messages yet. Say hello to the community."
- Direct (`kind: "direct"`) rendering stays exactly as-is (conditional per kind).

Done when: community room renders as a left-aligned author-row feed with room
header + member count; direct chat unchanged; one primary action (Send) per screen.

- Update the no-conversation EmptyState to community framing ("Your community
