<objective>
Add a small, self-contained rich-text renderer so chat message bodies display their
formatting (bold, italic, inline/fenced code, bullet & numbered lists with nesting,
headings, blockquotes, links) and preserve line breaks — then seed the kit with long,
formatted, and multi-length conversations to exercise it.

Today `Message`/`Bubble` and the production `ChatClient` both output `body` as a raw JSX
string: markdown syntax shows as literal characters and `\n` collapses to a single space
(`white-space: normal`). No markdown library exists in the repo and none should be added.

Purpose: Give the team a visual + automated way to validate long/formatted text rendering
across both bubble variants, with one renderer shared by the kit and production.
Output: `MessageBody` component (+ tests, stories, barrel export), two call-site wirings,
expanded kit seed data, and an updated kit "Full container" section.
</objective>

  <files>apps/web/components/chat/message-body/message-body.tsx, apps/web/components/chat/message-body/index.ts, apps/web/components/chat/message-body/message-body.test.tsx, apps/web/components/chat/message-body/message-body.stories.tsx, apps/web/components/chat/index.ts</files>
  <behavior>
    Parser (compact, line-by-line — NOT a full CommonMark engine) over `body: string`,
    returning React elements. Cover each case with a test in message-body.test.tsx:
    - Bold `**x**` → &lt;strong&gt;; italic `*x*` and `_x_` → &lt;em&gt;
    - Inline code `` `x` `` → &lt;code&gt; with font-mono
    - Fenced block ```` ```lang\n…\n``` ```` → &lt;pre&gt;&lt;code&gt; block, language label optional, no inline parsing inside
    - Headings `#`/`##`/`###` at line start → heading elements (sized via text tokens, NOT raw px)
    - Blockquote `> ` at line start → &lt;blockquote&gt;
    - Bullet list `- ` / `* ` and numbered list `1. ` → &lt;ul&gt;/&lt;ol&gt;&lt;li&gt;; nesting via leading indentation (2+ spaces)
    - Links `[text](url)` → &lt;a&gt; with href, target="_blank", rel="noopener noreferrer"
    - Plain multi-line text: single `\n` → &lt;br&gt;; blank line → new paragraph (line breaks preserved)
    - SECURITY: link href sanitized — only http/https/mailto allowed; javascript:/data:/other
      schemes are dropped (rendered as plain text or href removed). Test a `javascript:` URL is neutralized.
    - Empty/whitespace body → renders nothing (no crash).
  </behavior>
  <action>Create `MessageBody` as a named-export function component (NOT forwardRef — it is
not focusable; links inside are natively focusable). Props: `{ body: string; mine?: boolean; className?: string }`.

elements directly, NEVER use dangerouslySetInnerHTML (XSS). Use `cn()` for all conditional

component small and self-contained — no new npm dependency. Add `index.ts` re-exporting it,
