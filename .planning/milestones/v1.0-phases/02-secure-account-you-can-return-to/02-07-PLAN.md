<objective>
Close the UAT test 7 gap: entering a wrong password on /login must show a single, calm, in-place field-level message without the whole page appearing to re-render.

The diagnosis (`.planning/debug/login-wrong-password-full-rerender.md`) proved there is NO reload or navigation — the wrong-password path is fully client-side and the copy does render. The perceived "whole page rerenders" is a whole-page LAYOUT SHIFT: /login vertically centers its Card (`flex min-h-dvh items-center justify-center`) and the Input reserves no space for its message line, so setting `passwordError` grows the card ~30px and flexbox re-centers everything (heading/email jump up 14.8px, button/links jump down 14.8px, measured at 1440x900). Secondary defect: the copy is wired to Input's heavy tier-2 `error` prop instead of the tier-1 soft `notice` tone the UAT expects.

Purpose: Restore the phase-01 layout-stability contract ("no control changes size on state change") to the shared Input — which Button already honors via its overlay spinner — and render the wrong-password copy in the correct soft tone. This is the last blocker on UAT test 7 for phase 02.

Output: An Input component that reserves constant height for its message row (showing a message changes text, not geometry), and a /login page whose wrong-password feedback uses the tier-1 `notice` tone.
</objective>

Input component props (apps/web/components/ui/input.tsx):
```typescript
interface InputProps extends InputHTMLAttributes<HTMLInputElement> {
  label: string;      // always shown above the field
  hint?: string;      // shown only when no notice/error is set
  notice?: string;    // Tier 1 — regular weight, IconInfoCircle, text-notice; hidden if `error` is set
  error?: string;     // Tier 2 — semibold, IconAlertCircle, text-error, border-error border-2; wins over `notice`
}
```

Message-row precedence today (all three rendered CONDITIONALLY with `mt-2` and NO reserved space):
- hint  -> `<p className="mt-2 text-[14px] text-muted">` when `hint && !notice && !error`
- notice-> `<p className="mt-2 flex items-center gap-1.5 text-[14px] text-notice">` + IconInfoCircle size 20 when `!error && notice`
- error -> `<p className="mt-2 flex items-center gap-1.5 text-[14px] font-semibold text-error">` + IconAlertCircle size 20 when `error`

Layout-stability reference — Button (apps/web/components/ui/button.tsx):
The loading spinner is absolutely overlaid over a still-mounted `opacity-0` label so entering/leaving the loading state never resizes the button. Comment on the button: "Layout stability: no state change may alter the rendered size." Apply the SAME principle to Input's message row.

Design tokens (apps/web/app/globals.css @theme — reference via Tailwind utilities, never raw hex):
- `--color-notice` (text-notice): light-dark(oklch(0.40 0 0), oklch(0.80 0 0)) — tier 1, regular weight
- `--color-error`  (text-error):  light-dark(oklch(0.20 0 0), oklch(0.95 0 0)) — tier 2, semibold; monochrome, NOT red
- `--color-muted`  (text-muted):  hints
- `--size-control`: 56px control floor

Login wrong-password branch (apps/web/app/login/page.tsx):
On a non-`email_not_confirmed` sign-in error, `setPasswordError("That email and password don't match. Try again?")` runs, and the Password Input currently receives `error={passwordError || undefined}` (line 79). The transport-failure catch sets the SAME `passwordError` state with connection copy.

Input consumers and their message props (blast radius of the Input change — all get the reserved row):
- login: Email (none), Password (`error` -> becomes `notice` in this plan)
- signup: Email (`error`), Password (`hint`) — tier-2 error MUST stay tier-2
- reset-password: Password (`hint` + `error` toggle) — tier-2 error MUST stay tier-2
- forgot-password, expired-link: no message props (get a reserved blank row)
- page.tsx, kit: showcase pages (hint/notice/error demos)
