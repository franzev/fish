Alert analog = Input's tone-prop shape + Card's non-focusable-container export style. Target contract:
- `type AlertTone = "notice" | "error" | "success"`
- `interface AlertProps extends HTMLAttributes<HTMLDivElement> { tone: AlertTone; children: React.ReactNode }`
- `export function Alert({ tone, children, className, ...props })` — named export, NO forwardRef (non-focusable), matches Card
- toneConfig record maps tone → { Icon, className (border), messageClassName (weight) }:
  - notice: IconInfoCircle, `border-border-strong`, `font-normal`
  - error: IconAlertCircle, `border-error border-2`, `font-semibold`
  - success: IconCircleCheck, `border-border-strong`, `font-normal`
- Container: `flex items-start gap-2 rounded-control border bg-surface p-4` + toneConfig.className
- Icon: `size={20}`, `stroke={tone === "error" ? 2 : 1.75}`, `aria-hidden="true"`, `className="mt-0.5 shrink-0"`
- Message: `<p className={cn("text-[15px] text-body", messageClassName)}>{children}</p>`
- Doc-comment convention: `/** One-line, explains product-rule context */` directly above the export (match Card/Button)

Alert example copy (UI-SPEC Copywriting Contract):
- notice: "That doesn't look like an email yet. Check the spelling?"
- error: "Something needs your attention before you can continue."
- success: "You're all set."

/kit page (from Plans 01/02) already has: KitThemeToggle at top, Button section, Input section, Card section, Progress section. This plan ADDS: tokens (color swatch table), typography (Lexend/Fraunces at the 4 declared sizes: 17/14/20/32), icons (the 3 Tabler icons), and the Alert section (all 3 tones). Section pattern: `<section className="mb-10"><h2 className="mb-4 text-xl">{name}</h2>...</section>`. Data-driven arrays (`.map(...)`) for repetitive swatches/icons.

Declared type scale (UI-SPEC Typography): Body 17px/400 Lexend, Label 14px/400-500 Lexend, Heading 20px/600 Fraunces, Display 32px/600 Fraunces. Exactly 2 weights (400, 600).

Token roles to display in the tokens section (monochrome, from Plan 01 ladder): bg, surface, surface-2, border, border-strong, foreground, body, muted, primary, on-primary, notice, error, success.
