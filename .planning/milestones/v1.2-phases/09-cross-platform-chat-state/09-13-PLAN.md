<objective>
Harden the portable `@fish/core/chat-state` contract so three source-proven
defects (09-VERIFICATION probes) can no longer occur, and encode each as a
cross-platform fixture vector so native parity stays honest:

- WR-02: authoritative hydration must not delete unresolved local sends.
- WR-03: a late failure must never downgrade an already-sent message.
- WR-10: `getMessageSnippet` must count in Unicode code points, stay <= 96,
  and never split a surrogate pair.

Purpose: these are portable-lifecycle failures — a browser judgment cannot fix
them, and native clients would inherit the same bugs from the fixtures.
Output: reducer + selector changes, five new fixture vectors, an updated
fixture runner, and synchronized protocol + native notes. No web runtime file
and no new dependency (the `chat-state-boundary` test must stay green).
</objective>
