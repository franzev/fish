<objective>
Close WR-01: today the portable reducer restores the failed body to `composer.draft` (per protocol/fixture), but `use-chat-composer.ts` immediately calls `setDraft(..., "")` after a failure and clobbers it; worse, a delayed failure first overwrites a newer draft with the failed body and then clears both. Make failure recovery atomic and draft-safe — preserve a newer draft typed while a send was pending, otherwise restore the failed body — and remove the unconditional post-failure clear.

Purpose: CSTATE-06 requires "no lost drafts". The fix aligns the web hook and the component tests with the portable protocol and tightens the reducer so a newer edit is never lost.
Output: conditional restore in the reducer, a new parity fixture, updated protocol + fixtures-name list, the hook clobber removed, and component tests aligned + a delayed-failure regression added.
</objective>
