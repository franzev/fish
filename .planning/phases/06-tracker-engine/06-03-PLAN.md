<objective>
Ship the assignment command boundary and the coach review surface: the `assign-tracker` Edge Function (the first Edge Function in this repo that verifies JWT identity, re-checks coach↔client membership with a caller-scoped client using an explicit `coach_id = caller.id` filter, and performs a service-role write with `23505` idempotency), its `config.toml` registration, a seed/verify-rls live invocation proving the authorization logic, the extraction of the shared `formatAnswer` module, the read-only `CoachTrackerTimeline` appended to `/coach/clients/[id]`, and the final Phase 6 release gates.

The Edge Function is the one genuinely net-new authorization surface in this phase — `send-message` is copied only for its `Deno.serve`/method-guard/JSON-response shape, NEVER for its authorization logic (it has none). The coach timeline is a near-exact copy of `CoachOnboardingReview`, and the two now share ONE extracted answer formatter.

Purpose: A coach/seed assigns exactly one tracker via an authorized, server-derived command that a client can never self-invoke (the caller-scoped `coach_id` filter never matches a client's uid), and a coach reviews the client's entries read-only.
Output: A working, JWT-verified, membership-gated, idempotent `assign-tracker` function; a shared answer formatter; a read-only tracker timeline on the coach detail route; and a fully green release gate.
</objective>
