<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → saveTrackerEntryAction | Untrusted `{ fieldId, answer }` crosses here; the action re-verifies the session and re-validates the answer against the DB field config before the RPC. |
| Server Action → save_tracker_entry RPC | The RPC (Wave 1) re-derives client_id from auth.uid() and enforces the active-assignment gate; the action never sends identity or version. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-07 | Spoofing / EoP | saveTrackerEntryAction called while signed out or with a forged identity field | mitigate | getCurrentUser() re-check; strictObject payload rejects client_id/coach_id; identity from session + RLS/RPC only. |
| T-06-08 | Tampering | malformed answer submitted to bypass config constraints | mitigate | validateFieldAnswer against the DB-loaded field config before the RPC; RPC + pg_jsonschema are the DB backstop. |
| T-06-09 | Tampering (design invariant) | a percentage/streak leaking into the client UI | mitigate | Grep gate over apps/web/components/tracker + /tracker; MilestonePath renders shape-only states, no numeric label passed to Progress as judgement. |
| T-06-10 | Information Disclosure | XSS via malformed config rendered as HTML | mitigate | Config labels rendered as React text via FieldRenderer (never dangerouslySetInnerHTML); malformed config shows the calm notice state. |
| T-06-SC | Tampering | npm installs | accept | Zero new npm packages this plan; all UI primitives are reused from Phase 5. No install task. |
</threat_model>
