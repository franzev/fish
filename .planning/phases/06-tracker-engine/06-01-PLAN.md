<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client → tracker_entries write | A client may only write their own entry, only via `save_tracker_entry`, only against their active assignment. |
| authenticated user → tracker_assignments write | No authenticated write path exists; assignment is service-role-only (Wave 3 Edge Function). |
| service-role/seed → config tables | Trusted config authoring; guarded by pg_jsonschema CHECK + freeze triggers so even a trusted caller cannot persist malformed or retroactively-mutate a used version. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Elevation of Privilege | tracker_assignments direct INSERT by a client | mitigate | No authenticated INSERT policy; `checkTrackerSelfAssignRejected` proves the DB rejects it. |
| T-06-02 | Tampering | malformed tracker field config persisted | mitigate | zod (app) + `jsonb_matches_schema` DB CHECK (byte-for-byte from 0008); `checkTrackerMalformedConfigRejected`. |
