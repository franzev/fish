<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client browser -> Next Server Action -> SQL command | Untrusted POST/action input crosses into authenticated command writes. |
| Authenticated user -> Supabase RLS tables | JWT claims and `auth.uid()` decide row visibility and write ownership. |
| Seed/migration service role -> database | Service role bypasses RLS and is limited to migration/seed execution. |
| Coach review -> assigned client answers | Coach can see sensitive onboarding answers only through `private.is_coach_of`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing/Elevation | `save_onboarding_answer`, `finalize_onboarding_attempt` | mitigate | Functions derive `client_id` from `auth.uid()`, reject null `auth.uid()`, and expose no `client_id` or `coach_id` input. |
| T-05-02 | Elevation/Tampering | Service-role seed/migration boundary | mitigate | Service role appears only in `scripts/seed.ts` and migrations; app actions call authenticated RPC through user-scoped Supabase services. |
| T-05-03 | Information Disclosure | onboarding RLS policies | mitigate | Client own-read policies and coach policies compose `private.is_coach_of(client_id)`; `verify:rls` asserts unassigned and cross-client zero-row reads. |
| T-05-04 | Tampering/Repudiation | assessment versions/questions/config | mitigate | `pg_jsonschema` CHECK rejects malformed config; used-version/question triggers reject mutation after attempts exist. |
| T-05-05 | CSRF/Tampering | Server Action POSTs | mitigate | Later Server Actions re-check `getUser()` and zod-parse every payload; this plan keeps DB functions safe even if called directly. |
| T-05-06 | Tampering/Denial | autosave/finalize race | mitigate | Save command performs attempt creation, answer upsert, snapshot, and resume cursor update atomically; finalize locks post-submit writes. |
| T-05-07 | Information Disclosure | coach answer review | mitigate | Answer snapshots are RLS-scoped and read-only to assigned coaches; no direct coach update/delete grants. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs in this phase; package audit says existing dependencies only. Do not run `pnpm add`. |
</threat_model>
