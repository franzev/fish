<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Coach route params -> server data access | Untrusted client id route param selects an assigned client review. |
| Supabase RLS -> coach review UI | Sensitive answers flow from RLS-scoped rows into the read-only coach view. |
| Coach UI -> client privacy | Coach must not see personal prefs/consent or answers for unassigned clients. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing/Elevation | route param id | mitigate | Existing UUID guard and RLS-backed `getCoachClientDetailData` confirm assignment before fetching review data. |
| T-05-02 | Elevation/Tampering | service-role misuse | mitigate | Coach route uses normal server Supabase services, not service-role, and exposes no write actions. |
| T-05-03 | Information Disclosure | coach review read | mitigate | `getCoachClientOnboardingReviewData` composes RLS with `private.is_coach_of`; tests preserve same unavailable state for denied clients. |
| T-05-04 | Tampering | mutable version/config | mitigate | Review renders answer snapshots from Plan 05-01; used versions/questions are DB-frozen. |
| T-05-05 | CSRF/Tampering | coach review surface | accept | No write Server Action or form exists on coach review; route is read-only. |
| T-05-06 | Tampering/Denial | partial/final state | mitigate | Partial/submitted labels are read from attempt status; no coach mutation path exists. |
| T-05-07 | Information Disclosure | sensitive answer overexposure | mitigate | Component renders only prompt and answer snapshots; hides a11y prefs/consent and has no cross-client detail on denial. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs; reuse existing UI and test stack. |
</threat_model>
