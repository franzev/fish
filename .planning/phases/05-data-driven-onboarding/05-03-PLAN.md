<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Server Actions | Client POSTs untrusted answer payloads and finalize requests. |
| Server Actions -> SQL command functions | Validated action payload invokes authenticated RPC commands. |
| Server Component -> Supabase reads | Server data loads must let RLS scope client and coach data. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Spoofing/Elevation | `saveOnboardingAnswerAction`, SQL RPC | mitigate | Action re-checks `getCurrentUser()`; RPC derives `auth.uid()` and receives no client id. |
| T-05-02 | Elevation/Tampering | Service-role misuse | mitigate | App code uses `createServerSupabaseServices()` and authenticated RPC only; no service-role env vars in app files. |
| T-05-03 | Information Disclosure | `getClientOnboardingData`, RLS reads | mitigate | Repository reads through RLS; no manual cross-client filtering substitutes for policy. |
| T-05-04 | Tampering | malformed config persistence | mitigate | Action validates via zod and Plan 05-01 DB CHECK rejects malformed persisted config. |
| T-05-04a | Tampering | answer/config mismatch | mitigate | `saveOnboardingAnswerAction` loads the question config, calls `parseFieldConfig` and `validateFieldAnswer`, and blocks mismatched option ids, scale values, and text lengths before RPC. |
| T-05-05 | CSRF/Tampering | Server Action POSTs | mitigate | Each action re-checks session and parses with `saveOnboardingAnswerSchema`; tests assert direct no-session rejection. |
| T-05-06 | Tampering/Denial | autosave/finalize race | mitigate | RPC atomically upserts answer/resume state; UI preserves draft on failure and does not clear on rejected save. |
| T-05-07 | Information Disclosure | route rendering sensitive answers | mitigate | Client route renders only current user's RLS-scoped attempt; home only exposes start/resume state, not answers. |
| T-05-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs; reuse existing stack. |
</threat_model>
