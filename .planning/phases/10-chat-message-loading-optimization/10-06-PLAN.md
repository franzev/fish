## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local chat cache -> reconnect read | Client-held rows choose a read cursor, but only server-confirmed rows are authoritative enough to delimit the RLS-protected backfill query. |
| Authenticated SSR service -> Supabase | The server repository sends conversation- and message-id-scoped reaction selects under the caller's existing Supabase session; RLS remains the authorization boundary. |

## STRIDE Threat Register (ASVS L1; block on high)

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-10-14 | Tampering | Reconnect cursor selection | medium | mitigate | Accept only `localStatus === "sent"` rows as cursor material; otherwise reset through the bounded newest-window action. Exact-input tests prevent pending/failed client timestamps or ids from reaching the backfill action. |
| T-10-15 | Denial of Service | Initial reaction enrichment | medium | mitigate | Restrict reads to retained message ids, batch at 25 ids, keep 1000-row pagination per batch, and return without a query for an empty window. |
| T-10-16 | Information Disclosure | Reaction read filters | low | mitigate | Keep the existing `conversation_id` predicate and caller-session RLS; message ids narrow the result but never replace authorization. No service-role client or privileged function is introduced. |
| T-10-17 | Repudiation / consistency | Reconnect recovery | low | mitigate | Preserve server-authoritative `(created_at,id)` ordering, the existing merge/dedup reducer path, and bounded reset metadata; tests assert the exact marker used. |
