<objective>
Wire the portable pagination contract (Plan 01) and the read actions (Plan 02) into the web state
layer: the Zustand adapter gets pagination dispatch wrappers + selectors, the message hook gets a
guarded `loadOlderMessages` and a `applyGapBackfill`, initial hydration switches to `hydrateWindow`,
and the triplicated full-refetch-on-reconnect in the realtime hook is replaced by one coalesced,
bounded gap-backfill. This is the behavior "brain" — Plan 04 renders it.

Purpose: Make merge-in-place (CLOAD-02), single-path dedup (CLOAD-05), keyset load-earlier
(CLOAD-03), and reconnect backfill (CLOAD-06) real in the running app while keeping the CSTATE-03
web-only boundary intact.
Output: New store methods/selectors, message-hook callbacks, a consolidated reconnect path, and
extended store tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Realtime WS → client reducer | Realtime INSERT/UPDATE payloads (already RLS-filtered server-side) enter the client merge path. |
| Client → PostgREST (backfill read) | The reconnect backfill issues a bounded read; the session token must be fresh before it fires. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-06 | Tampering / Information Disclosure | Backfill fetch on a stale/expired token after wake-from-sleep | mitigate | Reuse the existing `subscribeAfterAuth` wait-for-fresh-session pattern before any backfill fetch (fails toward missing data, never toward other users' data; RLS applies per request regardless). |
| T-10-07 | Denial of Service | Reconnect storm triggering concurrent backfills | mitigate | Coalesce all three channels' reconnects behind one `backfillInFlightRef` lock and skip the first post-mount SUBSCRIBED, replacing three concurrent unbounded full-history refetches with at most one bounded backfill. |
| T-10-08 | Spoofing | Client store making an authorization decision | accept | The store/hooks make no auth/membership/assignment decision (CSTATE-03 boundary reasserted by the Task 1 boundary test); server RLS/Edge Functions remain authoritative. |
| T-10-SC | Tampering | npm/pnpm installs | accept | Zero new packages installed this plan. |
</threat_model>
