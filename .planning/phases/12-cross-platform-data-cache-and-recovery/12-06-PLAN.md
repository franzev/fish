<threat_model>
## Trust boundaries

| Boundary | Untrusted input crossing it | Required control |
|---|---|---|
| Verified auth owner → Room query | Owner and conversation selectors | Compound owner/conversation predicates; missing or mismatched owner yields zero |
| Accepted reducer result → Room transaction | Metadata rows, tombstones, cursors, history truth | Whole-batch ownership validation and one atomic transaction |
| Room database → feature hydration | Potentially stale or restored local rows | Verified-owner gate, schema version, safe DTO mapping, no local authorization |
| Cache limits/time → eviction | Clock values and row pressure | Injected clock, fixed tier ordering, protected newest 40 |
| Device backup/transfer → restored app | Private database files | `allowBackup=false`, extraction exclusions, verified identity before hydration |

## STRIDE threat register

| Threat ID | Category | Component | Disposition | Mitigation and verification |
|---|---|---|---|---|
| T-12-01 | Information disclosure | Private cached metadata | mitigate | Compound owner/conversation keys, verified-owner hydration, wrong-owner integration tests. |
| T-12-02 | Information disclosure | Memory-only delivery URLs | mitigate | Room schema has no URL lease/token/runtime-reference fields; PRAGMA and sentinel scans. |
| T-12-03 | Spoofing / elevation | Owner isolation | mitigate | Local rows never authorize; exact owner/conversation predicates; cached capabilities omitted. |
| T-12-04 | Tampering / information disclosure | Identity transition | mitigate | `purgeOwner` + `verifyOwnerPurged` are blocking primitives consumed by Plan 06; current plan proves isolation. |
| T-12-05 | Tampering / information disclosure | Filesystem/temp and backup | mitigate | App-private Room database, no-backup/device-transfer rules, no raw server-derived file deletion in this plan. |
| T-12-06 | Information disclosure / repudiation | Logs/diagnostics | mitigate | Store methods expose typed outcomes only and never accept diagnostic URL/path/byte strings. |
| T-12-07 | Tampering | Stale callbacks | mitigate | Only Plan 01 accepted owner/generation/request results may call atomic reconcile; repeated/stale transactions are idempotent. |
| T-12-08 | Denial of service | Cache growth | mitigate | 400/conversation, 2,000/account, 30-day inactivity, and protected-newest eviction tests. |
</threat_model>
