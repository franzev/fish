<threat_model>
## Trust boundaries

| Boundary | Untrusted input crossing it | Required control |
|---|---|---|
| Connectivity → fetch policy | Captive/unvalidated/data-saving network | VALIDATED gate and separate lookaheadAllowed |
| Signed delivery → memory/disk | URL/token and bytes | Memory-only registry, opaque keys, no response cache |
| Filesystem → owner purge | Generated paths | Standardized containment and absence verification |

## STRIDE threat register

| Threat ID | Category | Component | Disposition | Mitigation and verification |
|---|---|---|---|---|
| T-12-02 | Information disclosure | Delivery leases | mitigate | Registry-only storage and sentinel scan. |
| T-12-05 | Tampering / disclosure | Thumbnail files | mitigate | Opaque contained paths, atomic writes, purge proof. |
| T-12-06 | Information disclosure | Diagnostics | mitigate | Typed categories only. |
| T-12-08 | Denial of service | Fetch/cache | mitigate | 50-ID cap, 64 MiB/30-day pruning, Data Saver lookahead suppression. |
| T-12-SC | Tampering | Package supply chain | accept | No package install occurs. |
</threat_model>
