<threat_model>
## Trust boundaries

| Boundary | Untrusted input crossing it | Required control |
|---|---|---|
| Verified auth identity → portable cache hydration | Claimed owner, conversation, and persisted snapshot | Exact owner/conversation/generation match before any row becomes observable |
| Supabase page/realtime result → reducer | Provider rows, cursors, request identity, tombstones | Phase 11 whole-event ownership and request/cursor/replace gates plus generation |
| Visibility/lifecycle/network signals → recovery planner | Trigger bursts and arbitrary item IDs | Deduplication, at-most-50 chunks, one cycle, attempts 0/1 only |
| Delivery response → client state | Short-lived signed URLs and expiry | Memory-only lease contract; no cached snapshot or diagnostics field |
| Portable state → later presentation | Cache/recovery truth and provider failures | Closed vocabulary; provider strings, paths, URLs, counters, and byte limits excluded |

## STRIDE threat register

| Threat ID | Category | Component | Disposition | Mitigation and verification |
|---|---|---|---|---|
| T-12-01 | Information disclosure | Private cached metadata | mitigate | Explicit allowlisted snapshot; owner/conversation/generation gate; wrong-owner fixture yields zero rows. |
| T-12-02 | Information disclosure | Memory-only delivery URLs | mitigate | No delivery URL/token field in snapshot or presentation; secret-sentinel vector and structural field scan. |
| T-12-03 | Spoofing / elevation | Owner isolation | mitigate | Verified identity and conversation are mandatory; cached capabilities hydrate false; RLS remains authoritative. |
| T-12-04 | Tampering / information disclosure | Identity transition | mitigate | Generation changes reset state before new owner binding; A→B, A→nil, unresolved, and A→B→A vectors. |
| T-12-05 | Tampering | Filesystem and temp cleanup contract | mitigate | Portable contract exposes opaque references only to cleanup adapters and never accepts raw deletion paths; platform plans verify containment and absence. |
| T-12-06 | Information disclosure / repudiation | Logs and diagnostics | mitigate | Presentation and diagnostic projections accept only operation/outcome/category; token/path/provider-string sentinel is rejected. |
| T-12-07 | Tampering | Stale async callbacks | mitigate | Exact generation + owner + conversation + cycle/request/cursor/replace acceptance; stale events are complete no-ops. |
| T-12-08 | Denial of service | Fetch and recovery work | mitigate | Caps, 50-ID batching, data-saving suppression, 500ms coalescing, one in-flight cycle, and one automatic retry. |
</threat_model>
