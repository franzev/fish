<objective>
Close the single UAT blocker from test 3: clicking the email verification link 500s. The diagnosis (`.planning/debug/verify-email-link-500.md`) proved this is NOT a code bug — FISH's `/auth/confirm` handler, proxy, and env keys are all healthy and verified end-to-end on port 3001. The failure is an environment/config mismatch: `supabase/config.toml` pins `site_url` to `http://127.0.0.1:3000`, so `{{ .SiteURL }}` in the confirmation email renders a link at :3000 — but on this machine :3000 is permanently held by an unrelated project (Timberyard), and Next silently fell FISH back to :3001. The emailed link therefore hits the foreign app, which returns a raw 500.

The fix is surgical config/dev-script alignment (not a feature): make the FISH dev port and `site_url` agree deterministically. Pin the dev port explicitly so Next can never silently divert auth emails again, and point `site_url` + `additional_redirect_urls` at that same port.

Purpose: Make the last user-flow blocker green so UAT tests 4–13 can run. This is the recurrence of the known port wrinkle flagged in plan 02-05.

Output: an explicit dev port in `apps/web/package.json`; `site_url` + `additional_redirect_urls` in `supabase/config.toml` aligned to that port; a Supabase stack restart; and a human re-run of UAT test 3 with a FRESH signup.
</objective>
