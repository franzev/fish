<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Supabase Auth | Client-issued auth calls cross into GoTrue; the anon/publishable key is public by design |
| navigation → proxy.ts | Every request passes the session-refresh proxy; a dropped cookie here silently logs users out |
| build → env | Secrets (`SUPABASE_SERVICE_ROLE_KEY`) must never reach the client bundle |
| package registry → repo | Two new third-party packages enter the dependency tree |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | proxy.ts / server.ts session read | mitigate | Only `getUser()`/`getClaims()` server-side; grep-gate forbids `.getSession(` in server.ts/proxy.ts (Task 2 acceptance) |
| T-02-02 | Denial of Service | proxy.ts cookie refresh | mitigate | Single `NextResponse`, cookies written to request AND response, no logic between client creation and getClaims() (Pitfall 5, Task 2) |
| T-02-03 | Information Disclosure | env var layout | mitigate | `SUPABASE_SERVICE_ROLE_KEY` never `NEXT_PUBLIC_`-prefixed; `.env.local` git-ignored, `.env.example` values empty (Task 2) |
| T-02-SC | Tampering | npm installs (`@supabase/*`) | mitigate | Blocking human checkpoint verifies both [ASSUMED] packages on npmjs.com (official `supabase` org provenance) before `pnpm add`; slopcheck unavailable (no pip) — Task 1 |
</threat_model>
