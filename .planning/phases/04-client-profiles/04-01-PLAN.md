Column-scoped grant (level is NEVER named here — this is freeze layer 1):

  grant select on public.client_profiles to authenticated;
  grant update (
    goal, locale, timezone,
    theme_pref, text_size_pref, reduced_motion_pref,
    consented, consented_at, consent_version
  ) on public.client_profiles to authenticated;
  grant select, insert, update, delete on public.client_profiles to service_role;

Live-verified freeze behavior (this session, real HTTP through PostgREST):
- authenticated PATCHes only a granted column -> 200 OK
- authenticated PATCHes level (alone or mixed) -> 403, code 42501 (grant layer; trigger never runs; atomic, no partial write)
- service_role PATCHes level -> 200 OK (WHEN clause exempts it — seed path)
- if level is ever accidentally granted, the trigger independently rejects -> 400, code P0001

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client browser -> PostgREST/Supabase | An authenticated client (or a direct REST caller with a valid client JWT) can attempt any read/write; RLS + column grants are the only enforcement |
| service_role (seed) -> DB | Trusted; bypasses the freeze trigger's WHEN clause by design (seed sets level) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Elevation of Privilege | client attempts to UPDATE own `level` | mitigate | Column-scoped GRANT UPDATE never names `level` (403/42501) + BEFORE-UPDATE trigger `prevent_level_change` (P0001) — two independent layers, verified by `checkLevelFreezeRejected()` |
| T-04-02 | Information Disclosure | unassigned coach enumerates a client UUID | mitigate | Coach-read RLS policy uses `private.is_coach_of(id)`; non-assigned -> zero rows (no error, no side channel). Verified by `checkUnassignedCoachDenied()` |
| T-04-03 | Information Disclosure | client reads another client's row via direct REST | mitigate | self-read/self-update policies gate on `id = auth.uid()`; cross-client -> zero rows. Verified by `checkCrossClientDenied()` |
| T-04-04 | Tampering | future migration accidentally widens the `level` grant | mitigate | Trigger layer independently rejects `level` change even if granted (RESEARCH Pattern 1 case 5) |
| T-04-SC | Tampering | npm install of `zod` | mitigate | `zod@4.4.3` slopcheck `[OK]`, no postinstall, colinhacks/zod, ~6yr established (RESEARCH Package Legitimacy Audit) — legitimacy verified, no blocking checkpoint required |
