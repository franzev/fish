<objective>
Close WR-01: give the "load earlier" pagination path conversation/generation
ownership so a request started in conversation A can never affect conversation B
after the mounted client switches (09-VERIFICATION Plan 09-12 must-have; HV-02).

Two defects today (09-VERIFICATION "In-flight A request settling after switch to
B"): a single hook-wide `isLoadingOlderRef` boolean survives the switch (so A
gates B and A's reset clears B's lock), and the scroll-preserving wrapper
captures the viewport then later writes `scrollTop`/error state even if the
conversation already changed.

Purpose: this is a source-proven race, not a browser judgment. Output: a
per-conversation in-flight lock, a stale-completion guard keyed on the
`onLoadOlder` generation token, stale-rAF cancellation, and deferred-switch
tests that settle A as both success and failure after rerendering to B. Scoped
to the two hook files + the ChatClient test — no chat-client.tsx change.
</objective>
