<objective>
Close the round-5 UAT gap (Test 2, severity: minor): a failed older-page load
in `/channels/:id` fires TWO identical automatic attempts instead of one.

Root cause (confirmed in the debug session): the one-automatic-attempt gate is
split across two state systems that commit in separate renders. On failure,
`markOlderPageFailed` clears the store's `isLoadingOlder` (SyncLane via
useSyncExternalStore) one commit BEFORE the awaiting caller's local
`setHasOlderLoadError(true)` (use-load-older-messages.ts:64/91). In that
intermediate commit the observer guard `hasMoreOlder && !isLoadingOlder &&
!hasOlderLoadError` passes, the effect re-attaches the IntersectionObserver over
the still-intersecting sentinel, and the browser's initial observation fires a
second identical-cursor request. Bounded at two; 5/7 flaky by frame-timing race.

Purpose: the "one automatic attempt then a calm, manual retry" behavior is a
core chat-state correctness property (CSTATE-02/06) and the reason the failure
affordance reads as calm rather than as a silent double-fetch. Output: the
failure flag moves into the store's per-conversation pagination state so it
commits atomically with `isLoadingOlder=false`, the local error flag and the
callback-identity reset block are deleted, and a browser-faithful IO test mock
locks the fix against regression.

Fix direction is diagnose-confirmed, not exploratory — implement exactly the
three "missing" items from the UAT gap. Do not weaken or reshape any existing
behavior: the WR-01 deferred-switch tests and the bounded-retry / calm-affordance
/ manual-retry / conversation-reset tests must all stay green.
</objective>
