<objective>
Close the single blocker gap from 09-UAT.md Test 2: "loading earlier messages breaks the app."

A failed older-page load is left retryable by design at the portable state layer (reducer.ts `olderPageLoadFailed` — correct, do NOT change it), but the `IntersectionObserver` in `use-load-older-messages.ts` re-fires immediately once `isLoadingOlder` flips back to `false` because the top-of-list sentinel is still visible. With no delay, cap, or visible failure state, this is a tight, silent, unbounded retry loop — the tester read the flickering, unresponsive top-of-list as the app "breaking."

Purpose: Bound the auto-retry to a single attempt and give the user a calm, honest way to retry by hand, per the debug diagnosis fix direction.
Output: A gated sentinel observer, a load-outcome signal, a notice-tone failure affordance, and failure-path tests that would have caught this.

Full diagnosis: @.planning/debug/loading-earlier-messages-retry-storm.md
</objective>
