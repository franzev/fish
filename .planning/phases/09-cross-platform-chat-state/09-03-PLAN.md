<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser UI to Zustand store | Untrusted user interactions update local cache/coordination state only. |
| Zustand store to portable core | Store dispatches reducer events but does not own deterministic chat rules independently. |
| Store/hooks to server actions | Server actions, Edge Functions, database functions, and RLS remain authority for auth, assignment, membership, writes, persistence, and durable read state. |
| Test process to package registry | Only the audited `zustand` package is installed through pnpm into `apps/web`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-11 | Elevation of privilege | `chat-store.ts` | mitigate | Store tests/static assertions reject auth/session truth, role permission decisions, assignment logic, Supabase clients, and service-role data per D-08. |
| T-09-12 | Information disclosure | Global client store lifetime | mitigate | Store is browser-only, keyed by `conversationId`, reset in tests, and cleared/hydrated on conversation changes; no server component imports. |
| T-09-13 | Tampering | Optimistic send/retry store actions | mitigate | Store actions dispatch portable reducer events from Plan 09-01; duplicate `clientRequestId` behavior covered by fixture and store tests. |
| T-09-14 | Repudiation | Retry and confirmation events | mitigate | Retry reuses local `clientRequestId`; server action/RPC remains idempotent and authoritative for durable result. |
| T-09-15 | Denial of service | Selector subscriptions | mitigate | `chat-selectors.ts` returns narrow slices by `conversationId`; `ChatClient` does not subscribe to the entire store. |
| T-09-SC | Tampering | pnpm install of `zustand` | mitigate | RESEARCH.md package legitimacy audit marks `zustand` OK; install uses `pnpm --filter @fish/web add zustand` only, with lockfile update reviewed. |
</threat_model>
