<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Web route to portable core | Untrusted browser events are transformed into deterministic reducer events, but the reducer is not an authorization or persistence authority. |
| Portable core to native contract | JSON fixtures define behavior that Android/iOS can replay without importing web state libraries. |
| Client state to Supabase/server | Supabase RLS, Edge Functions, server actions, and database functions remain the authority for auth, assignment, membership, writes, and durable read state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Elevation of privilege | `packages/core/src/chat-state/*` | mitigate | Reducer/types cannot contain auth/session/role permission or assignment decisions; CSTATE-01 boundary test rejects web/server authority imports. |
| T-09-02 | Information disclosure | Fixture and state shapes | mitigate | Fixtures use synthetic IDs/names only and do not contain seeded account credentials, JWTs, service-role data, or real message contents. |
| T-09-03 | Tampering | Optimistic merge/retry reducer | mitigate | `confirmSentMessage` and `mergeRemoteMessage` reconcile by `id`, incoming `clientRequestId`, and explicit local request id; fixture cases cover duplicate request reconciliation. |
| T-09-04 | Repudiation | Send failure/confirmation results | mitigate | Fixture cases preserve failed local message body/status and confirm exact result transitions; durable send idempotency remains in Supabase per D-05. |
| T-09-05 | Information disclosure | Web compatibility shim | mitigate | Shim imports portable selectors only and does not import Supabase clients, server actions, session data, or Zustand. |
| T-09-SC | Tampering | npm/pnpm installs | accept | This plan performs no package-manager install; package install is isolated to Plan 09-03 and covered by the approved RESEARCH.md package audit. |
</threat_model>
