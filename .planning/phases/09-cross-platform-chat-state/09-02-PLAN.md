<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser UI to web hooks | User input enters local state and server-action calls; hooks must not become authorization or persistence authorities. |
| Web hooks to Supabase adapters | Realtime and server actions remain web-specific and never move into `packages/core`. |
| Server actions to Edge Functions/RLS | Existing command boundary continues to verify auth and membership outside React local state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Elevation of privilege | `use-chat-composer.ts` and `use-chat-read-state.ts` | mitigate | Hooks receive server actions as dependencies and never decide membership, role permission, or persistence; D-05 authority remains in server/Supabase. |
| T-09-07 | Information disclosure | Hook local state lifetime | mitigate | Hooks are scoped to mounted `ChatClient` and one authorized `chat.conversationId`; no module-level message arrays or cross-user globals. |
| T-09-08 | Tampering | Realtime merge callbacks | mitigate | Callbacks route incoming messages/read states through portable-backed merge helpers from Plan 09-01. |
| T-09-09 | Denial of service | Realtime/presence subscriptions | mitigate | Extracted effects return cleanup functions that unsubscribe/remove channels and stop presence sessions on unmount/conversation change. |
| T-09-10 | Repudiation | Retry/edit/delete/reaction commands | mitigate | Composer hook calls existing server actions with the same payload fields and does not bypass command validation in `actions.ts`. |
| T-09-SC | Tampering | npm/pnpm installs | accept | This plan performs no package-manager install; package install is isolated to Plan 09-03 and covered by the approved RESEARCH.md package audit. |
</threat_model>
