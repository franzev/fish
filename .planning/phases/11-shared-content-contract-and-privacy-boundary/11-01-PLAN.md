<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| authenticated caller -> listing/category/context RPCs | Untrusted UUIDs, categories, limits, and cursors cross into private conversation data. |
| normalized RPC -> source tables | A secure projection must not rely on client filtering or bypass child-table RLS. |
| sender delete command -> attachment cleanup queue | One authorized message tombstone fans out to every derived item and durable Storage work. |
| cleanup RPC -> Edge worker | Service-role-only claims and tokens cross into destructive Storage operations. |
| persisted link job -> gallery metadata | Only an already validated first public URL may become canonical; enrichment failure must not change safety. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-01 | Information disclosure | listing/category/context RPCs | mitigate | SECURITY INVOKER, `private.is_conversation_member`, RLS, generic missing/unauthorized results, and cross-conversation tests. |
| T-11-02 | Elevation of privilege | function grants and search path | mitigate | Empty search path, fully-qualified objects, revoke public/anon, authenticated-only reads, service-role-only cleanup; never make listing SECURITY DEFINER. |
| T-11-03 | Information disclosure | links and child tables | mitigate | Persist only `firstPublicHttpUrl`, harden deleted-source RLS, exclude provider/export and delivery URLs, and test direct table access. |
