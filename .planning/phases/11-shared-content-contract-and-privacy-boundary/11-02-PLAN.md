<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| validated body text -> canonical link tables | A safe first public URL becomes private gallery metadata before third-party enrichment. |
| authenticated/service clients -> verifier | Tests must exercise real RLS identities without leaking service credentials into application paths. |
| cleanup secret -> Edge worker | A secret-authorized destructive worker claims rows and removes private objects. |
| database claim -> Storage API -> finish RPC | Partial failures must remain retryable and must not mark undeleted objects complete. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-01 | Information disclosure | direct table/RPC access | mitigate | Real signed-out/member/outsider/other-conversation clients test RLS, invoker membership, grants, deleted-source policies, and generic results. |
| T-11-02 | Elevation of privilege | cleanup and listing RPC grants | mitigate | Plan 05 asserts anon/authenticated denial for cleanup and anon denial for listing; this plan calls cleanup only through service-role clients. |
| T-11-03 | Information disclosure / SSRF | canonical link persistence | mitigate | Reuse `firstPublicHttpUrl`; write only canonical URL/hostname; never persist unsafe URL or logs; failed enrichment retains only safe identity. |
| T-11-04 | Tampering | cursor/query optimization | mitigate | Exact identity vectors, reference-query comparison, complete cursor validation, and no unmeasured index/branch-limit optimization. |
| T-11-05 | Tampering / denial of service | deletion cleanup | mitigate | Secret check, service-only durable claims, SKIP LOCKED, release-on-sibling-failure, idempotent finish, missing-object success. |
| T-11-06 | Information disclosure | delivery URLs and logs | mitigate | Verifier captures behavior in memory only; helpers never persist/log URL, path, token, body, or private metadata. |
| T-11-07 | Information disclosure | identity purge contract | transfer | Plan 03 defines and tests purge-before-expose; Phase 12 owns device persistence and temp-file deletion. |
| T-11-SC | Tampering | package-manager supply chain | mitigate | No package installs; lockfiles stay unchanged. Unexpected installs require slopcheck and blocking human legitimacy review. |
</threat_model>
