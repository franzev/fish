## Trust Boundaries

| Boundary | Description |
|---|---|
| publishable-key identities -> RLS/RPC | Untrusted callers probe conversation UUIDs, tables, cursors, categories, and functions. |
| service-role setup -> adversarial assertions | Administrative seeding must not accidentally authorize the code path under test. |
| database rows -> generated TypeScript | Generated/provider shape drift must not become a handwritten application contract. |
| verifier output -> logs/summary | Evidence must be useful without revealing private metadata, credentials, URLs, or Storage paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-01 | Information disclosure | conversation/list/category/context access | mitigate | Real signed-out/outsider/other/former/member identities, cross-conversation cursor probes, direct-table checks, and generic result assertions. |
| T-11-02 | Elevation of privilege | RPC privilege/grant boundary | mitigate | Anonymous execute denial, member-only invoker reads, service-only cleanup, and generated signature/grant verification. |
| T-11-03 | Information disclosure / SSRF | safe-link path | mitigate | Validate canonical first-public URL persistence and unsafe zero-write behavior; test deleted link direct access and avoid logging URLs. |
| T-11-04 | Tampering | order, sentinel, and cursor | mitigate | Exact multi-page identities, complete/tampered/cross-conversation cursors, tie cases, reference query, and cursor from row 40. |
| T-11-05 | Tampering / denial of service | deletion fan-out/cleanup | mitigate | Sender/recipient/repeat cases, new-access denial, sibling fan-out, claim concurrency/retry/missing-object convergence. |
| T-11-06 | Information disclosure | identity purge | transfer | Plan 03/04 fixtures prove purge-before-expose; Phase 12 owns persistent cache/temporary-file implementation. |
| T-11-07 | Information disclosure | signed URLs and verifier logs | mitigate | In-memory capture only, no values printed/persisted, explicit pre-issued TTL nuance, stable safe PASS/FAIL output. |
| T-11-SC | Tampering | package-manager supply chain | mitigate | No installs or lockfile changes; unexpected dependencies require slopcheck and blocking legitimacy review. |
