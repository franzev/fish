<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| canonical JSON -> Android resource loader | Kotlin must decode exact wire fields without maintaining local expected semantics. |
| canonical JSON -> copied SwiftPM resource | Copy drift must fail before Swift tests can claim parity. |
| prior identity -> next identity | Both reducers must purge all private metadata/delivery/temp references before exposure. |
| stale page/realtime -> deleted source | Native state must reject resurrection even when async data arrives after deletion. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-01 | Information disclosure | native authorization assumptions | mitigate | Parity tests cover permission results but explicitly rely on server RLS/RPC; no local member filter is treated as authority. |
| T-11-02 | Information disclosure | identity purge | mitigate | Both native reducers replay purge-before-expose vectors that clear metadata, delivery, decoded, temp, cursor, and error state. |
| T-11-03 | Tampering | stale page resurrection | mitigate | Source tombstone guards, sibling fan-out, repeated tombstone, and stale-page vectors in both runtimes. |
| T-11-04 | Tampering | order/cursor drift | mitigate | Preserve decoded server order and exact cursor/sentinel contract; never reproduce locale-dependent sorting. |
| T-11-05 | Information disclosure / legal boundary | GIF/sticker export | mitigate | Native capability types decode and assert `canExport=false`; UI implementation remains out of this phase. |
| T-11-06 | Tampering | fixture divergence | mitigate | Android consumes canonical core resources directly; iOS uses byte-identical sync plus blocking `--check`. |
| T-11-07 | Elevation of privilege | RPC grants | transfer | Plan 01 owns grants and Plan 02 adversarially verifies them; native code receives no service-role or cleanup capability. |
| T-11-SC | Tampering | package-manager supply chain | mitigate | No dependencies are added; Gradle/SwiftPM/pnpm locks stay unchanged and unexpected installs block for legitimacy review. |
</threat_model>
