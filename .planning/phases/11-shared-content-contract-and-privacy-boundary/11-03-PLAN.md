<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| database-normalized row -> portable client state | The client receives private metadata but must not treat local filtering as authorization. |
| account A -> account B | Identity-bound in-memory/delivery/temp references must be purged before another verified owner is exposed. |
| old page/realtime event -> tombstoned state | Stale asynchronous data must not resurrect deleted content. |
| canonical JSON -> three runtime implementations | Fixture drift or local reinterpretation could silently split permissions/order semantics. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-01 | Information disclosure | client authorization assumptions | mitigate | Types/tests state that database RLS/RPC is authority; fixtures never substitute for malicious-caller tests. |
| T-11-02 | Information disclosure | identity change | mitigate | `identityChanged` synchronously purges every identity-bound field before new content can merge. |
| T-11-03 | Tampering | stale page after deletion | mitigate | Persistent deleted-source guard, source-message fan-out, idempotent tombstones, and resurrection vectors. |
| T-11-04 | Tampering | order/pagination drift | mitigate | Server order preserved, stable ID dedupe, exact cursor/sentinel vectors, codepoint comparison only for reference fixtures. |
| T-11-05 | Information disclosure | item DTO | mitigate | Exclude signed URLs, Storage staging paths, claims/tokens, scan details, hashes, upload credentials, and diagnostics. |
| T-11-06 | Information disclosure / legal boundary | GIF/sticker capability | mitigate | Contract and fixtures fix `canExport=false` until rights are independently verified. |
| T-11-07 | Elevation of privilege | package exports | mitigate | Pure core exports only; architecture tests reject provider/framework imports and no raw database contract is re-exported. |
| T-11-SC | Tampering | package-manager supply chain | mitigate | No package installs or lockfile edits; unexpected additions require legitimacy review. |
</threat_model>
