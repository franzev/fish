## Trust Boundaries

| Boundary | Description |
|---|---|
| event envelope -> item payload | A correctly addressed event can still carry another conversation's private item. |
| async request -> completion | Late, duplicate, or reordered completions can overwrite newer continuation state. |
| canonical JSON -> reducer tests | Silent fixture drops can make insecure behavior appear portable. |
| tombstone/identity change -> pending request | Old asynchronous work must not resurrect deleted or prior-owner data. |
