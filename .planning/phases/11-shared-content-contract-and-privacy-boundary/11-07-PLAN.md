## Trust Boundaries

| Boundary | Description |
|---|---|
| message body -> canonical link row | Syntactically plausible hostnames are untrusted until A/AAAA answers are all public. |
| DNS answer -> canonical proof | Every returned address must be public before version-2 proof is written. |
| canonical/legacy job -> network boundary | No outbound preview request is permitted, so redirects and rebinding cannot reach a socket. |
| legacy enrichment job -> canonical row | Compatibility draining may change only job state and must never mutate the canonical row. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-11-07-01 | Information disclosure / SSRF | DNS and canonical proof | mitigate | Validate the nonempty A/AAAA union, reject any malformed/private/reserved result, and write no proof on failure. |
| T-11-07-02 | Information disclosure / SSRF | outbound preview/redirect path | mitigate | Remove outbound metadata fetching; unit and hosted cases require zero network requests, so redirects/rebinding cannot reach private services. |
