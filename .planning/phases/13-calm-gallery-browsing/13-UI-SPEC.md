# Phase 13 — UI Design Contract

> Visual and interaction contract for calm gallery browsing on Android and iOS. Generated by gsd-ui-researcher; the checker upgrades `status` after verification.

---

## Phase UI Boundary

Phase 13 adds one full-screen, per-conversation **Shared content** destination to the Android and iOS direct-chat apps. Clients and coaches reach it from:

1. one dedicated conversation-header control; or
2. one full-width row in conversation details.

The screen is a quiet retrieval surface, not a content-management surface. It renders only categories that contain accepted items, gives each category an appropriate scanning layout, preserves useful cached content through recovery, and exposes one bounded request for older history.

Phase 13 does **not** add preview, autoplay, export, share, save, download, source-message navigation, deletion, search, sorting, filters, favorites, albums, bulk selection, new sending, a global gallery, a mobile dashboard, or a web gallery. Item components expose one provider-neutral `SelectItem(itemId)` seam for Phase 14, but Phase 13 must not ship an enabled no-op control or imply that preview already exists.

Sources: `13-CONTEXT.md`, `REQUIREMENTS.md` DISC-01/DISC-02, `ROADMAP.md`, and the native direct-chat-only boundary in `AGENTS.md`.

## Spacing Scale

### Policy resolution: existing FISH tokens govern

Phase 13 introduces zero phase-local spacing values. The generic 8-point example is subordinate to the repository's implemented semantic scale. The allowed subset below consists entirely of existing generated values and uses multiples of four:

| Existing token role | Generated value | Usage |
|---------------------|-----------------|-------|
| `twoXs` | 4dp / 4pt | Media-grid gap, compact icon-to-label separation, selected-tab indicator offset |
| `xs` | 8dp / 8pt | Top-bar inset, tab inner spacing, compact row subgroups |
| `sm` | 12dp / 12pt | Row icon-to-copy gap, title-to-metadata gap, state content rhythm |
| `md` | 16dp / 16pt | Row block padding and notice internal rhythm |
| `page` | 20dp / 20pt | List/state horizontal screen inset and gallery notice inset |
| `lg` | 24dp / 24pt | State-to-content and category-to-content separation |
| `xl` | 40dp / 40pt | Empty/unavailable state separation |
| `twoXl` | 48dp / 48pt | Major state-region separation only |

Exceptions are existing component/geometry metrics, not spacing:

- Android interactive controls use the existing 48dp `touchTarget`; iOS controls use the existing 44pt `targetTouch`.
- Media uses an 88dp/pt semantic minimum cell width normally and 120dp/pt at accessibility text sizes. These are gallery content dimensions, not margin/gap values.
- Existing 1dp/pt dividers, 2dp/pt selected-tab indicators, 4dp/pt `chatInner` radius, 12dp/pt `control` radius, and 16dp/pt `card` radius remain component-owned.
- No one-off numeric padding, margin, gap, or `Spacer` value is allowed. Layout code selects the equivalent existing platform token by semantic role.

Sources: `AGENTS.md` spacing discipline; `docs/ui-ux-agent-guidelines.md` precedence and resolved grid rule; generated Android/iOS tokens.

## Typography

Phase-owned gallery content uses exactly four existing sizes and two weights. Existing shared components retain their internal token mapping; feature code must not apply raw font sizes.

| Existing token role | Size | Weight | Line height | Phase 13 semantic use |
|---------------------|------|--------|-------------|-----------------------|
| `caption` | 13sp / 13pt | Regular 400 | 1.4 | File type/size, link hostname, voice duration, supporting boundary copy |
| `ui` | 15sp / 15pt | Regular 400 | 1.5 | Category tabs, item titles, quiet actions |
| `body` | 17sp / 17pt | Regular 400 | 1.55 | Empty/unavailable explanations |
| `heading` | 20sp / 20pt | Semibold 600 | 1.15 | Full-screen and empty/unavailable headings |

Rules:

- All copy uses sentence case.
- Screen title and empty-state heading use Fraunces; browsing content uses Lexend.
- Category selection is not communicated through a third weight. Use the selected indicator, foreground contrast, and programmatic selected state.
- Item titles wrap to two lines at normal sizes and grow without a line cap at accessibility sizes. Secondary metadata remains one logical line when possible but may wrap; it must never overlap or be clipped.
- Filenames and hostnames use natural text direction/isolation so punctuation remains readable in RTL layouts.
- Dynamic Type/font scaling is mandatory on both platforms. Do not shrink type to preserve a column count.

## Color

Use the generated semantic platform tokens rather than raw values.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Android `FishTheme.colors.background`; iOS `Palette.bg` | Full-screen gallery canvas |
| Secondary (30%) | Android `surface`/`surfaceAlt`; iOS `Palette.surface`/`surface2` | Top bar, notices, row pressed state, skeletons, missing-thumbnail tiles |
| Accent (10% maximum) | Android `primary`, `foreground`, `selected`; iOS `Palette.primary`, `foreground`, `surface3` | Selected-tab indicator/text, visible focus, and the existing monochrome busy indicator only |
| Required text | `foreground` for headings/titles; `body` for explanation and metadata; `muted` only for truly secondary non-actionable metadata |
| Notice | Android `notice`; iOS `Palette.notice` | The single Phase 12 cache/recovery notice icon and boundary only |
| Destructive | Existing `error` token, unused | Reserved for Phase 14 destructive action only; Phase 13 has no destructive action |

Accent is reserved for the selected category, visible focus, and active progress/busy feedback. It is not used to color categories, decorate thumbnails, badge items, tint file types, or make every row bright.

Cached, stale, incomplete, unavailable, selected, loading, and focused states must each have a non-color cue. Offline and ordinary recovery failures use the approved calm notice treatment, never an alarming red item or full-screen wash.

## Information Architecture and Navigation

### Conversation header entry

- Add one quiet photo-outline icon button with accessible label `Shared content`.
- Logical action order is existing audio call, video call, message search, **Shared content**, then existing account content where present.
- Do not add a badge, count, tooltip dependency, primary fill, label chip, or overflow menu.
- The participant identity cluster remains the conversation-details path. It is a separate target and must not be merged with Shared content.
- Targets never shrink below the platform design-system metric. At constrained widths, hide the visual presence subtitle first, then truncate the participant name; preserve the combined participant identity accessibility label. Do not shrink or overlap actions.

### Conversation details entry

- Add one full-width quiet navigation row immediately after the participant identity group and before friend-safety actions.
- Row anatomy: 20dp/pt photo-outline icon, `Shared content` in `ui`, flexible spacer, directional chevron.
- Minimum row height: Android 48dp; iOS 44pt. Use `Spacing.md` horizontal content padding and `Radius.control`.
- The row uses the normal surface/pressed surface. It is not a primary button, warning, card, count, or badge.
- Android extends the existing `ParticipantDetailsSheet`. iOS adds the equivalent native conversation-details sheet and makes the participant identity cluster a button labeled `Conversation details`.

### Full-screen destination

- Present Shared content as a full-screen native destination on the conversation navigation stack. It is never a sheet, dialog, details subsection, or global route.
- Title: `Shared content`.
- Back from a header-origin gallery returns to the conversation and restores focus to the header Shared content control.
- Back from a details-origin gallery returns to conversation details, with focus restored to its Shared content row; dismissing details then returns to the conversation.
- The entry origin is route state, not inferred from visual state. System back gestures and Back controls produce the same result.
- Selected category and its scroll state survive child destinations while Shared content remains on the stack. They are session-only and are not persisted across app launches.

## Category Contract

### Population and order

- Canonical order is **Media, Files, Links, Voice**.
- Derive the visible category set from accepted, display-eligible items only.
- Render no control for empty categories.
- If exactly one category is populated, remove the entire category-control slot; do not leave blank reserved height.
- On first open, select the first populated category in canonical order.
- If accepted realtime/refresh data adds another category, keep the current selection and add the new tab in canonical order.
- If an accepted tombstone/removal empties the selected category, select the first remaining populated category in canonical order and announce the new selection politely. If no categories remain after an authoritative accepted update, render the authoritative empty state.

### Visual and semantic treatment

- Use plain text tabs with one 2dp/pt bottom indicator for the selected tab.
- Keep every tab target at least 48dp Android / 44pt iOS high.
- Tabs use `ui` type and equal visual priority. No icons, pills, segmented-control capsules, counts, badges, or category colors.
- Android uses the Material primary text-tab primitive styled with FISH tokens.
- iOS uses a feature-owned text-tab row rather than the system segmented picker, whose filled capsule conflicts with the locked plain-text treatment.
- At accessibility text sizes, the tab row scrolls horizontally rather than clipping or shrinking labels. Bring the newly selected tab fully into view. Preserve canonical logical order and native RTL behavior.
- Expose the category group as one tab list/adjustable peer-control group and each tab as selected or not selected. Announce `{Category}, selected` only on an actual selection change.

## Gallery Layout Contract

The selected-category content is the screen's primary visual anchor: the Media grid when Media is selected, or the Files, Links, or Voice list when its category is selected. It receives the largest continuous area and the strongest repeated visual rhythm. Native navigation orients, tabs clarify scope, notices explain exceptional state, and pagination extends history; all four remain visually subordinate and must not compete with the selected content through primary fill, oversized type, extra containment, or persistent emphasis.

### Shared content frame

Reading order is:

1. native top bar;
2. optional category tabs;
3. at most one Phase 12 gallery-level notice;
4. selected-category content;
5. retained-history boundary and its one optional action.

The notice uses `Spacing.page` horizontal inset and `Spacing.xs` vertical separation. It never overlays content. Accepted cached items remain at normal opacity and remain browsable while refresh/retry occurs.

### Media

- Use an adaptive square `LazyVerticalGrid`/`LazyVGrid`.
- Normal minimum cell width: 88dp/pt. Accessibility-text minimum: 120dp/pt. Maximum six columns.
- Use `Spacing.twoXs` between cells and `Spacing.page` at screen edges.
- Tiles are 1:1, use stable `itemId` keys, `Radius.chatInner`, and no card/shadow.
- Photos, videos, and GIFs use aspect-fill crop. Stickers use aspect-fit on `surfaceAlt` so transparent artwork is not clipped.
- A video receives one small noninteractive play glyph at the bottom trailing edge; a GIF receives one compact `GIF` text marker. Photos and stickers have no decorative overlay.
- Missing or failed thumbnails keep the tile geometry and show a tonal placeholder plus the item-kind glyph. They do not collapse, become blank, or show a retry button per tile.
- Animated GIFs and videos do not autoplay. Grid thumbnails are static.

### Files

- Use one single-column row per item; no nested cards.
- Row anatomy: file-text icon in a fixed touch-sized leading slot, filename, then `{localized type} · {localized file size}`.
- Use the safe original filename; fallback title is `File`.
- Derive a concise localized type from the safe extension/MIME type (for example `PDF` or `Document`). Never show a raw MIME string when a friendly type exists.
- Format byte size with the native locale-aware file-size formatter.

### Links

- Use one single-column row per item.
- Row anatomy: link icon in a fixed leading slot, safe preview title, then canonical hostname.
- Title fallback is the hostname. Do not show the raw URL, description, site favicon, tracking query, or a second action.
- Isolate hostname direction in RTL so dots and labels remain readable.

### Voice

- Use one single-column row per item.
- Row anatomy: microphone icon in a fixed leading slot, `Voice message`, then localized duration.
- Format duration as `m:ss` below one hour and `h:mm:ss` at one hour or more, using locale-appropriate digits.
- Do not render play/pause, waveform, speed, scrubber, or autoplay in Phase 13.

### Shared row behavior

- Files, Links, and Voice rows are at least 64dp/pt tall at default text sizes and grow vertically for wrapped/large text.
- Use `Spacing.page` horizontal screen inset, `Spacing.md` row block padding, `Spacing.sm` icon-to-copy gap, and one existing divider between rows; omit the divider after the last item.
- The entire row/tile is the future single selection target. No nested button, chevron, checkbox, menu, long-press action, swipe action, or multi-select affordance.
- Component APIs expose only `SelectItem(itemId)`. Phase 13 production integration must not attach an enabled no-op action. Phase 14 enables the target when its native preview destination exists.
- Pressed and focus geometry must not shift.

### Voice-duration data contract gap

The current Kotlin/Swift/TypeScript shared-content item projection has no duration field, while the locked Phase 13 decision requires Voice rows to show duration. Phase 13 must resolve this through a safe provider-neutral `durationMs` metadata field carried by the authoritative shared-content projection, cache DTOs, fixtures, and gallery UI model.

- `durationMs` is a non-negative integer or absent for legacy content.
- Do not persist or display delivery URLs to obtain it.
- Do not fetch or decode full audio merely to discover duration.
- A legacy item without trusted duration renders `Duration unavailable` as required body text. Never show `0:00`, `--:--`, or a fabricated estimate.

## Loading, Recovery, and Pagination

### Initial loading

- With no eligible cache, show structure-matched skeletons below the top bar and any known category control.
- Media skeleton count is exactly three computed grid rows (`columnCount × 3`, maximum 18).
- Files, Links, and Voice show exactly six row skeletons with a leading-square slot and two text bars.
- Skeletons are silent to assistive technology. Expose one parent loading status: `Loading shared content`.
- Use the existing calm opacity pulse. Under reduced motion, skeletons remain static.

### Cached and stale content

Consume the approved Phase 12 copy and truth without variation:

- refresh with cache: `Checking for updates…`;
- offline with cache: `You're offline` — `This content is saved on this device and may be out of date.`;
- exhausted refresh with cache: `Content may be out of date` — `We couldn't check for updates.` plus one `Try again`.

Keep cached items visible and normally interactive beneath the one notice. Do not dim content, show per-item badges, reset the selected category, jump scroll, or replace content with skeletons.

### Earlier history

- At an online incomplete boundary, show one quiet secondary control labeled `Show earlier content`.
- The action is centered or full-width within a bounded boundary slot, never primary-filled.
- On activation, keep the selected category and visible anchor stable; disable duplicate activation and expose busy semantics. Label remains `Show earlier content`; add the native busy indicator without changing target width.
- Append only an accepted page. Use stable keys and scroll-anchor preservation so content does not jump.
- If the request fails, retain all visible content and show `Earlier content didn't load. Try again.` with one quiet `Try again` action in the same boundary slot.
- At an offline incomplete boundary, show the approved Phase 12 text `Connect to see more shared content.` and no inert action.
- When history is complete, remove the boundary slot entirely.
- Never auto-load history at scroll end. Each older page requires one explicit activation.

The older-page request is global to the conversation even when the selected page contains no new items for the current category. Keep the current category; do not switch categories or claim that category history is complete until the shared continuation contract says all history is complete.

## State and Copy Matrix

Copy keys and truth conditions match across Android and iOS. Platform punctuation may vary only when grammar requires it; the terms `Shared content`, `Media`, `Files`, `Links`, and `Voice` do not vary.

| State | Visible content | Copy | Action | Focus/announcement |
|-------|-----------------|------|--------|--------------------|
| Initial online, no cache | Structure-matched skeletons | `Loading shared content` | None | Polite status once; focus stays on Back |
| Authoritative content | Selected populated category | No state copy | Future item selection only | Category then content in reading order |
| One populated category | Content only; no tab slot | No category copy | None | Screen title flows directly to notice/content |
| Refresh with cache | Cached items at normal opacity | `Checking for updates…` | None | No focus movement; announce only if perceptible |
| Offline with cache | Cached items at normal opacity | `You're offline` — `This content is saved on this device and may be out of date.` | None | Polite once |
| Exhausted refresh with cache | Cached items at normal opacity | `Content may be out of date` — `We couldn't check for updates.` | `Try again` | Preserve item/category focus |
| Authoritative empty | No item/category control | `No shared content yet` — `Items shared in this conversation will appear here.` | None | Announce after loading completes |
| Offline, no eligible cache | No item/category control | `Shared content isn't available offline` — `Connect to the internet to load it.` | None; reconnect is automatic | Announce once |
| Identity ineligible/resolving | No prior-owner item/category content | Host resolving state; never reveal prior-cache copy | None | Do not announce private prior-owner state |
| Authority unavailable, retry allowed | No item/category content | `Shared content isn't available right now` — `Reconnect and try again.` | `Try again` only when the Phase 12 contract enables it | Focus remains/restores on retry |
| Online incomplete history | Current content | Boundary action only | `Show earlier content` | Boundary follows last accepted item |
| Loading earlier history | Current content | Native busy status | Disabled `Show earlier content` with busy indicator | Preserve selected category and scroll anchor |
| Earlier-page failure | Current content | `Earlier content didn't load. Try again.` | `Try again` | Focus stays on retry |
| Offline incomplete history | Current content | `Connect to see more shared content.` | None | Read as boundary text, not alert |
| Legacy voice duration absent | Voice row remains visible | `Duration unavailable` | None | Include in the row's accessible description |

`Empty` is legal only after an authoritative successful response confirms zero eligible items. Offline/no-cache, identity-ineligible, loading, permission/authority failure, and a selected category with no accepted items must not map to `empty`.

## Accessibility Contract

- Every header/details entry has the exact accessible name `Shared content`.
- The photo icon, row-kind icons, media overlays, and notice icons are decorative when the enclosing control already has a complete label.
- Category controls expose role, label, and selected state. Loading controls expose busy state and prevent duplicate activation.
- Item labels are descriptive and type-specific:
  - Media: `{Photo|Video|GIF|Sticker}: {safe name or safe title}`; omit the colon suffix when unavailable.
  - File: `{filename}, {friendly type}, {localized size}`.
  - Link: `{safe title}, {hostname}`.
  - Voice: `Voice message, {localized duration}` or `Voice message, duration unavailable`.
- Use collection semantics so TalkBack/VoiceOver can announce item position without placing visible counts or badges in the UI.
- Focus order follows visual order: Back, title/header action if any, categories, notice/action, content, retained-history boundary.
- Switching category moves accessibility focus to the selected tab, then allows ordinary forward traversal into content. It does not force focus to the first item.
- Refresh and realtime reconciliation do not steal focus. If a focused item is authoritatively removed, move focus to the next item, previous item, selected category, or empty-state heading in that order.
- Retain focus on `Show earlier content` while it becomes busy and after failure. After accepted append, keep focus at the boundary control unless the platform's native collection semantics require the first newly appended item.
- Support large text without clipping, overlap, fixed-height rows, or shrinking targets. Media adapts column count; list rows grow.
- RTL mirrors directional chrome/chevrons and row geometry while preserving server order. Filenames and hostnames use text-direction isolation.
- Meaning never relies on color. Selected tabs use indicator plus semantic state; unavailable/loading states use copy and semantics.
- Use no Phase 13 animation beyond native stack continuity, pressed state, busy indicators, and the existing skeleton pulse. Reduced motion sets custom durations to zero/static.

Phase 15 owns final paired-platform TalkBack/VoiceOver, large-text, RTL, contrast, reduced-motion, performance, and release proof. Phase 13 must implement the semantic hooks and geometry that make those gates pass.

## Interaction and State-Holding Contract

- Gallery state is keyed by verified owner identity plus conversation ID.
- UI state holds visible populated categories, selected category, one scroll state per visible category, the Phase 12 presentation contract, and the older-page boundary state.
- Provider details remain behind Android `:data:chat` and iOS `ChatData`.
- Opening the destination calls the existing `SharedContentStore.open()` once per actual route entry; recomposition/body recalculation must not start duplicate recovery.
- Visible-item reporting uses stable item IDs and sends the actual visible set plus one-screen lookahead through the Phase 12 bounded delivery planner.
- Category switching reports the newly visible category set; it does not restart the recovery cycle or reset pagination.
- Accepted refresh/realtime changes preserve the user's visible anchor where possible. New newest items may be inserted above without forcing the user to the top.
- Returning from a child route restores selected category, scroll position, and focused item while the gallery route remains on the stack.
- Leaving/popping Shared content clears session-only category/scroll/focus state. Reopening selects the first populated category again.
- No UI event can bypass owner/conversation/generation/request/cursor acceptance gates.

## Cross-Platform Parity Rules

- Android and iOS use the same entry labels, category order, populated-only rule, first-selection rule, copy keys, item information hierarchy, state truth, pagination action, and focus-restoration intent.
- Native presentation may differ only where platform conventions require it: Compose Material text tabs versus a SwiftUI text-tab row, platform Back gestures, native busy indicators, and locale formatters.
- Both platforms use the same safe provider-neutral gallery UI model and the same TypeScript-owned shared-content fixtures for category/item projection.
- Both platforms preserve server order. Neither locale-sorts items or categories.
- Neither platform introduces an extra menu, swipe gesture, long press, item badge, per-item retry, refresh control, or settings choice.
- Snapshot/preview fixtures cover the same semantic matrix: light, dark, RTL, accessibility text, loading, one category, four categories, cached, stale, authoritative empty, offline unavailable, older-page busy/failure, and every item kind.

## Prohibited UI

Phase 13 must not add:

- a global or cross-conversation gallery;
- a mobile home/dashboard, lesson, assigned-work, community, marketplace, or web surface;
- search, sorting, filters, favorites, albums, counts, badges, or bulk/multi-select;
- item overflow, long-press, context, swipe, export, share, save, download, source-message, or delete actions;
- inline audio/video playback, GIF animation, autoplay, waveform, scrubber, or preview;
- a floating action button, primary CTA, bottom action bar, modal gallery, or category picker menu;
- per-item cached/stale/offline/loading badges or blanket dimming;
- automatic older-page loading or unbounded infinite scroll;
- raw provider errors, URLs, paths, MIME strings, private IDs, or diagnostics;
- any new sending, capture, upload, MIME-expansion, transcoding, OCR, AI, or summary pipeline.
