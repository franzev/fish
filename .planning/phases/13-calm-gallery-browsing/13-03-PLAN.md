<threat_model>
## Trust Boundaries

This plan explicitly tests conversation/identity binding, stale callbacks, the private delivery/cache boundary, and cross-conversation navigation.

| Boundary | Description |
|----------|-------------|
| verified owner/conversation → model | Tests require exact coordinator eligibility before accepted items become visible. |
| Swift Task callback → route/store | Generation, request, cursor, and route lifetime reject stale completion. |
| Core Data/delivery → SwiftUI model | Only safe DTOs may cross; URLs, paths, provider errors, and bytes remain private. |
| NavigationStack conversation path → Shared content | Explicit origin and conversation ownership prevent cross-conversation reuse. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | model/store tests | mitigate | Require `isGalleryEligible(for:)` and exact owner/conversation before any projection. |
| T-13-02 | Tampering | async store/model callbacks | mitigate | Tests revoke generation on pop/sign-out/switch and reject wrong request/cursor append. |
| T-13-03 | Information disclosure | Core Data, snapshots, accessibility | mitigate | Secret sentinels assert no delivery URL/path/raw owner/provider value crosses durable or view APIs. |
| T-13-04 | Elevation / Information disclosure | NavigationStack destinations | mitigate | Explicit `.header`/`.details` origins and conversation switch tests prevent stale route reuse. |
| T-13-SC | Tampering | dependency supply chain | accept | Existing SwiftPM dependencies only; no package or resolution change is allowed. |
</threat_model>
