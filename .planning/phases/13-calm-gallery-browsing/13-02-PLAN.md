## Trust Boundaries

This plan explicitly tests conversation/identity binding, stale callbacks, the private delivery/cache boundary, and cross-conversation navigation.

| Boundary | Description |
|----------|-------------|
| verified owner/conversation → presenter | Tests require exact owner and conversation before displaying accepted items. |
| coroutine callback → store/session | Old generation, request, cursor, or route callbacks must be ignored. |
| Room/delivery layer → Compose model | Only safe metadata and opaque item IDs may cross; URLs and paths stay private. |
| conversation stack → Shared content route | Explicit origin and conversation ID prevent cross-conversation route reuse. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | presenter/store tests | mitigate | Assert verified-owner and exact-conversation binding before any category or item is visible. |
| T-13-02 | Tampering | coroutine/page callbacks | mitigate | Adversarial tests reject stale generation/request/cursor/replace completions and duplicate earlier taps. |
| T-13-03 | Information disclosure | screenshot/semantics/cache projection | mitigate | Sentinel scans assert no raw owner IDs, delivery URLs, paths, provider errors, or content bytes. |
| T-13-04 | Elevation / Information disclosure | `ChatRoute` navigation | mitigate | Test both explicit origins and a conversation switch while gallery work is in flight. |
| T-13-SC | Tampering | dependency supply chain | accept | No dependency or lockfile change is allowed; tests use existing Gradle/Compose infrastructure. |
</threat_model>
