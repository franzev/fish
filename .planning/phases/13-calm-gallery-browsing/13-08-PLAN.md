<threat_model>
## Trust Boundaries

This plan enforces conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation/session clearing on iOS.

| Boundary | Description |
|----------|-------------|
| coordinator eligibility → store binding | The active verified owner/generation must match before any accepted item publishes. |
| Swift Task completion → store/model | Exact generation/request/cursor/route lifetime reject stale callbacks. |
| ChatData DTO → PersonalChat model | Only safe display metadata crosses; private delivery/cache values remain internal. |
| route-scoped model → conversation | Session anchors and selection are cleared on pop/switch to prevent cross-conversation reuse. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | coordinator/store attach | mitigate | Attach only after verified eligibility and revoke before purge/bind; exact owner/generation checked on every callback. |
| T-13-02 | Tampering | async append/recovery | mitigate | One in-flight global token and exact cursor/mode checks; close/revoke cancels tasks and advances generation. |
| T-13-03 | Information disclosure | gallery model | mitigate | Safe display allowlist excludes URLs, paths, provider errors, bytes, and action authority. |
| T-13-04 | Information disclosure | session anchors | mitigate | Route-scoped owner/conversation model is cleared on pop and replacement. |
| T-13-SC | Tampering | dependency supply chain | accept | No new package; Foundation/SwiftUI formatters and existing dependencies only. |
</threat_model>
