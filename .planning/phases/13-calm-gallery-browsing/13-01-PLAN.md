## Trust Boundaries

This plan explicitly covers conversation/identity binding, stale callbacks, the private delivery/cache boundary, and cross-conversation navigation/projection even though it creates tests rather than a native route.

| Boundary | Description |
|----------|-------------|
| verified identity → conversation RPC | Tests must prove the caller is a current member before rows are returned. |
| async request token → accepted fixture/page | Owner, conversation, generation, request, cursor, and append mode remain exact. |
| provider/cache data → test evidence | Private delivery/cache values must not enter fixtures, logs, or summaries. |
| conversation route → gallery projection | Test vectors must reject mixed-conversation items before category projection. |
