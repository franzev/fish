<threat_model>
## Trust Boundaries

This plan keeps conversation/identity binding, stale callbacks, the private delivery/cache boundary, and cross-conversation navigation/session state explicit at the Android UI boundary.

| Boundary | Description |
|----------|-------------|
| conversation/identity presenter → screen | The screen may render only the active verified owner/conversation projection. |
| Compose effect → presenter callback | Visibility and display confirmation may fire after recomposition or route disposal. |
| private ChatData → UI | Delivery/cache locators must never enter strings, semantics, logs, or screenshots. |
| navigation session → gallery anchors | Category/focus/scroll state must not survive into another conversation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | screen binding | mitigate | Accept one owner/conversation-bound presenter from composition and render no independent repository state. |
| T-13-02 | Tampering | Compose lifecycle callbacks | mitigate | Key effects by route generation and item key; dispose observers before any later confirmation can publish. |
| T-13-03 | Information disclosure | rows/semantics/snapshots | mitigate | Render the safe display allowlist only; exclude signed URLs, paths, provider errors, bytes, and cache locators. |
| T-13-04 | Information disclosure | remembered UI state | mitigate | Keep session state in the route presenter and clear it on pop/switch; never save cross-conversation anchors. |
| T-13-SC | Tampering | dependency supply chain | accept | No package install; existing Compose, Material, and Roborazzi dependencies only. |
</threat_model>
