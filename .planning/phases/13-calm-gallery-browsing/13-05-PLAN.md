<threat_model>
## Trust Boundaries

This plan preserves conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation safety in Android data contracts.

| Boundary | Description |
|----------|-------------|
| Supabase JSON → strict DTO | All 29 fields, conversation ownership, category/kind, duration, and order are validated before use. |
| accepted request → Room transaction | Owner/generation/request/cursor/replace must still match before cache mutation. |
| Room cache → gallery consumer | Cache supplies safe metadata only and never grants membership or delivery authority. |
| active conversation → hydrated namespace | Exact owner and conversation compound keys prevent cross-conversation reads. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | `DefaultChatRepository` | mitigate | Preserve verified-owner membership and identity-generation acceptance before hydrate/reconcile. |
| T-13-02 | Tampering | strict DTO and Room migration | mitigate | Reject malformed/negative duration and stale request/cursor/mode before any transaction. |
| T-13-03 | Information disclosure | Room v10 | mitigate | Persist only safe nullable metadata; schema/sentinel tests forbid URLs, paths, tokens, bytes, and capabilities. |
| T-13-04 | Information disclosure | cache namespace | mitigate | Compound owner/conversation predicates and whole-page validation reject mixed routes/pages. |
| T-13-SC | Tampering | dependency supply chain | accept | No package install; existing Room/Supabase versions and lockfiles remain unchanged. |
</threat_model>
