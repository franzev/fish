## Trust Boundaries

This plan requires evidence for conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation clearing on both platforms.

| Boundary | Description |
|----------|-------------|
| verified owner/conversation → native route | Evidence must prove only an eligible exact conversation binds and renders. |

| private repository/cache → UI/evidence artifacts | Tests, semantics, logs, and screenshots must not disclose delivery/cache internals. |
