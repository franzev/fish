<threat_model>
## Trust Boundaries

This plan enforces conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation clearing in Android production composition.

| Boundary | Description |
|----------|-------------|
| verified auth/conversation → live composition | A store may bind only after exact eligibility and conversation identity are known. |
| route/lifecycle callback → store generation | Back, reconnect, realtime, focus, and display callbacks may arrive after replacement. |
| ChatData delivery/cache → feature UI | Private locators and display-confirmation authority remain behind the data module. |
| explicit navigation origin → restored destination | A wrong/reused origin could reveal another conversation or restore the wrong surface. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | MainActivity/ChatRoute binding | mitigate | Construct only from verified owner plus active conversation and reject every identity/generation mismatch. |
| T-13-02 | Tampering | route/store callbacks | mitigate | Exactly-once open, one current request token, synchronous revoke/close, and stale callback rejection. |
| T-13-03 | Information disclosure | ChatDataModule boundary | mitigate | Expose safe models and a narrow confirm capability; never expose signed delivery/cache values to feature code. |
| T-13-04 | Information disclosure | origin/back stack | mitigate | Carry an explicit owner/conversation-scoped origin and clear destination/session state on pop or replacement. |
| T-13-SC | Tampering | dependency supply chain | accept | No package install; existing Android navigation/Compose/data dependencies only. |
</threat_model>
