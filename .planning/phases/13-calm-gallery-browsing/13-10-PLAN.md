<threat_model>
## Trust Boundaries

This plan keeps conversation/identity binding, stale callbacks, the private delivery/cache boundary, and cross-conversation navigation/session state explicit at the iOS UI boundary.

| Boundary | Description |
|----------|-------------|
| conversation/identity model → view | SwiftUI may display only the active verified owner/conversation projection. |
| SwiftUI task/effect → model callback | Async visibility/display callbacks may outlive the destination. |
| private ChatData → PersonalChat/FishUI | Delivery/cache locators must never enter labels, accessibility, logs, or snapshots. |
| NavigationStack session → model anchors | Category/focus/scroll state must not transfer to another conversation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | model injection | mitigate | Receive one route owner/conversation-bound model; never fetch independently from the view. |
| T-13-02 | Tampering | SwiftUI tasks/effects | mitigate | Key callbacks by current route generation and item; cancel/discard them on disappearance. |
| T-13-03 | Information disclosure | row/accessibility/snapshot output | mitigate | Render only safe display models; exclude signed URLs, paths, bytes, cache keys, and provider errors. |
| T-13-04 | Information disclosure | navigation/model state | mitigate | Route-scoped model clears anchors/selection on pop and identity replacement. |
| T-13-SC | Tampering | dependency supply chain | accept | No package install; SwiftUI, Foundation, FishUI, and existing snapshot tooling only. |
</threat_model>
