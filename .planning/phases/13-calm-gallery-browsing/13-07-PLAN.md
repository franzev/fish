<threat_model>
## Trust Boundaries

This plan enforces conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation/session clearing on Android.

| Boundary | Description |
|----------|-------------|
| verified store binding → accepted items | Only current owner/conversation/generation data may publish. |
| async refresh/append → store state | Exact cycle/request/cursor/mode gates reject stale callbacks. |
| store safe DTO → presenter labels | Provider, delivery, cache, and private identifiers remain hidden. |
| route session → conversation state | Presenter session is keyed and cleared on pop/switch to prevent cross-conversation reuse. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | `SharedContentStore.bind` | mitigate | Require verified owner and exact conversation; publish nothing while identity is unresolved/ineligible. |
| T-13-02 | Tampering | `loadEarlier` and recovery callbacks | mitigate | Match owner/generation/cycle/request/cursor/replace and suppress duplicate in-flight requests. |
| T-13-03 | Information disclosure | presenter/display model | mitigate | Allowlist safe metadata and localized labels; exclude URLs, paths, provider failures, bytes, and action authority. |
| T-13-04 | Information disclosure | presenter session | mitigate | Key and clear selection/anchors by conversation route lifetime; mixed-conversation inputs are rejected upstream. |
| T-13-SC | Tampering | dependency supply chain | accept | No dependency change; existing Kotlin/Compose/platform formatters only. |
</threat_model>
