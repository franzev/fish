<threat_model>
## Trust Boundaries

This plan preserves conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation safety for every native consumer of the shared contract.

| Boundary | Description |
|----------|-------------|
| verified identity → hardened RPC | Migration must retain membership/RLS authority and exact conversation predicates. |
| database row → portable projection | Duration/category values are validated before entering shared product state. |
| provider row → generated contract | Generated types prevent handwritten field-shape drift. |
| conversation item set → category projection | Mixed-conversation data must never be projected by native callers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing / Elevation | migration 0063 RPC | mitigate | Copy the complete hardened 0062 membership checks and run authenticated former-member tests. |
| T-13-02 | Tampering | `duration_ms` and projection | mitigate | Database check plus non-negative-integer portable validation; preserve request/cursor acceptance unchanged. |
| T-13-03 | Information disclosure | generated/safe contracts | mitigate | Add duration only; do not expose delivery URLs, storage paths, provider errors, bytes, or full audio. |
| T-13-04 | Information disclosure | `projectSharedContentGallery` | mitigate | Function consumes accepted conversation-owned items only; adversarial mixed-conversation tests remain upstream and native. |
| T-13-SC | Tampering | dependency supply chain | accept | No dependency install or lockfile edit; use existing Supabase CLI and TypeScript toolchain. |
</threat_model>
