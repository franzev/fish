<threat_model>
## Trust Boundaries

This plan preserves conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation safety in iOS data contracts.

| Boundary | Description |
|----------|-------------|
| Supabase JSON → strict Swift wire | Exact row shape, owner/conversation, order, and duration are validated before domain mapping. |
| accepted request → Core Data transaction | Verified identity and exact token/cursor/mode gate durable mutation. |
| Core Data → PersonalChat | Only safe Codable metadata crosses; cache never authorizes access. |
| active conversation → cache predicates | Exact owner/conversation predicates prevent namespace crossover. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | `SupabaseSharedContentRepository` | mitigate | Preserve verified-owner and authorized-directory checks before any page/cache acceptance. |
| T-13-02 | Tampering | strict decoder/Core Data | mitigate | Reject negative/malformed duration and stale request/cursor/mode before transaction. |
| T-13-03 | Information disclosure | Core Data model | mitigate | Persist only safe duration metadata; protection, backup, and secret-sentinel tests remain required. |
| T-13-04 | Information disclosure | owner/conversation cache | mitigate | Exact predicates plus whole-page mixed-conversation rejection prevent cross-route disclosure. |
| T-13-SC | Tampering | dependency supply chain | accept | No SwiftPM dependency or resolution change; platform Core Data and existing packages only. |
</threat_model>
