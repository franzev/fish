<threat_model>
## Trust Boundaries

This plan enforces conversation/identity binding, stale callback rejection, the private delivery/cache boundary, and cross-conversation navigation clearing in iOS production composition.

| Boundary | Description |
|----------|-------------|
| verified auth/conversation → App composition | Repository/store/model may bind only after exact verified owner and conversation eligibility. |
| NavigationStack/Task callback → store generation | Pop, scene, realtime, visibility, and display callbacks may outlive their destination. |
| ChatData delivery/cache → PersonalChat | Signed locators/cache details and confirmation authority remain private. |
| explicit navigation origin → restored destination | An old origin/path must not restore another owner or conversation. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-13-01 | Spoofing | FishApp composition | mitigate | Build from verified owner plus active conversation only and check identity/generation on every publication. |
| T-13-02 | Tampering | NavigationStack/Swift tasks | mitigate | Exactly-once open, synchronous revoke/close, cancellation, and request/cursor/destination token checks. |
| T-13-03 | Information disclosure | ChatData→PersonalChat | mitigate | Pass safe models/intents only; keep signed URLs, paths, bytes, and cache locators inside repository/store ports. |
| T-13-04 | Information disclosure | navigation path/origin | mitigate | Scope route values to owner/conversation, remove them on switch/pop, and clear route model state before restoration. |
| T-13-SC | Tampering | dependency supply chain | accept | No package install; existing SwiftUI, Core Data, Supabase, and FishKit dependencies only. |
</threat_model>
